# Query Contracts

Language-agnostic SQL query contracts belong here.

Use table-specific folders (for example `log/`, `order/`, `strength/`) and keep
queries named and stable so generated bindings can be produced for TypeScript,
Python, C#, and R clients.

## Current contracts

- `log/insert_log.sql`
- `log/select_logs.sql`
- `order/insert_order.sql`
- `order/select_orders.sql`
- `strength/select_strength.sql`
